# Voice Calibration

## Purpose

This file records the user's preferred fiction texture so the skill does not have to rediscover it every time.

Use this reference when planning, drafting, revising, or polishing prose for SheepStory.

## Core Preference

The user prefers fiction with:

- concrete scene texture
- strong visual continuity
- hard-SF / military-SF friction
- technical reasoning that changes decisions
- dry humor under pressure
- characters who are competent but not perfect
- worldbuilding revealed through objects, action, cost, scarcity, and longing
- intelligent characters whose reasoning can be reconstructed
- conflict that is not smoothed away too quickly

## Likes

- first-person or close-POV technical reasoning
- practical calculations, material limits, equipment constraints, and measurements
- crisis humor that does not erase danger
- scene openings that start from body, object, room, tool, vehicle, or immediate action
- close-to-wide movement from local detail to vast scale
- characters noticing anomalies and testing assumptions
- forgotten formulas recovered through derivation
- safety margins, uncertainty, and practical improvisation
- worldbuilding emerging from daily inconvenience
- military or engineering systems shown through what they cannot do
- dialogue with intention, obstacle, and subtext
- high-intelligence plans constrained by bad resources

## Avoid

- story-external encyclopedia worldbuilding
- Foundation-style historical inserts unless explicitly requested
- decorative science lectures
- textbook explanations with no scene consequence
- generic destiny / fate / chosen-one fog
- perfect genius characters
- perfect plans
- enemies becoming stupid to make the protagonist look smart
- polite customer-service dialogue
- every conflict being resolved by healthy communication
- over-polished prose that loses roughness
- AI-like symmetry in paragraphs, beats, or emotional summaries

## Preferred Technical Voice

When technical explanation appears, it should follow this energy:

```text
problem → observation → rule/principle → estimate/test → uncertainty → decision → human reaction
```

The explanation should feel like a person under pressure using knowledge as a tool, not a textbook giving a lesson.

## Preferred Scene Texture

When a scene needs atmosphere, use:

```text
body → object → room / vehicle / street → window / hatch / screen / scope → wider world → desire or denial
```

A setting detail should usually do at least two jobs:

- reveal world
- reveal character
- create friction
- change action
- show longing
- signal class/status
- create tension

## Preferred Intelligence Portrayal

Smart characters should:

- notice something specific
- test assumptions
- generate options
- reject a tempting plan
- choose a costly tradeoff
- update after failure

They should not simply “already know everything.”

## Calibration Prompt

When unsure about style, ask internally:

```text
Does this feel like lived-in hard-SF fiction with pressure, detail, and character voice, or does it feel like a clean AI summary?
```

If it feels like a clean AI summary, revise toward scene texture, friction, technical consequence, and voice.
