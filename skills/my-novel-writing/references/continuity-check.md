# Continuity Check

## Purpose

Continuity check prevents the story from resetting itself. It protects plot state, emotional consequences, character knowledge, object state, timeline, promises, and payoffs.

## Pre-Draft Continuity Gate

Before drafting prose, reconstruct the current story state.

Required fields:

```markdown
## Current Story State

### Last Known Position
- Last chapter / scene:
- Current location:
- Current time:
- Active POV:

### Character State
| Character | Physical state | Emotional state | Relationship changes | Knows / does not know |
|---|---|---|---|---|

### Object State
| Object | Owner | Location | Status | Last changed in |
|---|---|---|---|---|

### Open Threads
- Unresolved mysteries:
- Promises / foreshadowing:
- Pending consequences:
- Relationship tensions:
- Political / social pressure:

### Forbidden Contradictions
- Facts that must not change:
- Things the POV cannot know:
- Injuries, deaths, losses, debts, or betrayals that must still matter:
```

If any required field cannot be reconstructed, do not draft. Ask targeted continuity questions instead.

## Common Failure Patterns

Reject or flag these:

- A dead, absent, injured, captured, or exhausted character acts normally without explanation.
- A character knows information they never learned.
- A relationship softens without a scene earning the change.
- A destroyed, lost, or transferred object reappears casually.
- A mystery is answered before it is planted.
- A payoff happens before setup.
- A traumatic event has no emotional echo.
- The timeline allows impossible travel, recovery, or communication.
- A faction forgets what it publicly promised or threatened.

## Post-Draft Continuity Pass

After drafting, check:

1. Did every permanent change remain permanent?
2. Did the POV only know what the scene allowed?
3. Did emotional consequences survive into the scene?
4. Did any object or secret move without tracking?
5. Did any promise, question, clue, or foreshadowing change state?
6. Did the scene create new continuity obligations?

## Output Shape for Problems

```markdown
| Location | Continuity issue | Evidence | Fix |
|---|---|---|---|
```

## Principle

A novel is not a sequence of isolated scenes. Every scene inherits debt from the scenes before it.
