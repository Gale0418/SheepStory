# Story Foundation

## Purpose

Build an approved World Book and core Character Cards before substantial planning or prose begins. Start from whatever creative seed the user actually has instead of forcing every story through the same questionnaire.

Use this reference only when starting a new story or project without an approved Foundation. Skip it for continuation, revision, an already approved Foundation, or a sufficiently specified Quick standalone request.

## Foundation Status

Treat Foundation material as one of:

- `proposed`: generated for discussion but not canon
- `approved`: explicitly selected or confirmed by the user
- `superseded`: replaced by a later approved decision

Within a proposal, classify facts as `verified`, `inferred`, or `unknown`:

- `verified`: explicitly supplied or approved by the user
- `inferred`: a reversible interpretation awaiting confirmation
- `unknown`: materially unresolved

Never promote a proposal or inference to canon silently.

## Detect the Strongest Seed

Choose the entry path that preserves the most specific user material. Do not ask for information already supplied.

### World-First

Use when the user already knows the setting, era, social order, physical rule, speculative premise, or another world condition.

Ask first about the missing rule, cost, enforcement mechanism, lived consequence, or pressure that most affects possible characters. Do not ask the user to choose a generic world type again.

Derive character possibilities from how people survive, benefit, resist, enforce, misunderstand, or pay for the supplied world conditions. Keep all derived characters proposed.

### Character-First

Use when the user has a protagonist, relationship, voice, contradiction, occupation, wound, or desire but no settled world.

Ask first about the character pressure that should drive the story: what they want, what they cannot tolerate, what contradiction traps them, or what cost they refuse.

Only then offer world candidates. Make each candidate challenge the supplied character differently; do not attach a preferred genre merely because it is familiar.

### Situation / Image / Theme

Use when the seed is an event, scene image, relationship moment, question, theme, mood, object, or ending.

Identify what must remain recognizable in the seed. When several interpretations are plausible, offer two or three distinct Foundation candidates. For each candidate, show only:

- world premise
- central character pressure
- story engine
- main tradeoff
- meaningful difference from the other candidates

Do not bury the user under complete World Books before they choose a direction.

### No Seed

Use when the user wants a story but has no meaningful preference yet.

Ask one high-leverage question at a time. Start with whichever choice will eliminate the most incompatible directions, such as desired reading experience, central pressure, scale, or relationship focus. Offer a compact set of contrasting options plus a free-form answer.

Do not ask a full intake questionnaire. Do not generate an approved world or cast from silence.

## Progressive Questioning

Follow these rules:

1. ask one question at a time
2. explain why the choice matters only when it is not obvious
3. prefer choices that change several downstream decisions
4. reuse previous answers instead of asking the same thing in different words
5. stop asking when a coherent Foundation can be proposed
6. mark remaining non-blocking gaps as `unknown`

If the user prefers exploration, generate two or three candidates instead of interrogating them. If the user supplies a correction, update the proposal rather than defending the model's version.

## World Book

Capture only story-bearing world information:

```markdown
## World Book

### Premise / Genre / Scale

### Time and Place

### Divergence From Ordinary Reality

### Rules / Limits / Costs

### Power and Institutions

### Lived Consequences

### Resources / Scarcity / Enforcement

### Knowledge Boundary

- Disputed:
- Misunderstood:

### Prohibited Elements

### Canon Status

- Inferred:
- Unknown:
```

Exclude encyclopedia detail that does not constrain action, character, conflict, choice, or continuity. A World Book is a pressure model, not a trivia collection.

## Character Cards

Create a card only for characters important enough to affect choices or persistent state.

```markdown
## Character Card

### Function and Agency

### Public Face

### Private Want

### Fear / Wound

### Contradiction

### Strengths / Limits / Blind Spots

### Unacceptable Cost

### Knowledge / Misunderstanding / Secrets

### Voice

### Relationships / Leverage

### Current Pressure

### Change Hypothesis

### Canon Status
- Verified:
- Inferred:
- Unknown:
```

Treat the change direction as a hypothesis until the user approves it. Do not assign hidden identities, romance, betrayal, death, redemption, corruption, or final arcs without user evidence or approval.

## Cross-Check

Before presenting the Foundation, verify:

- world conditions create specific choices or pressure for the characters
- character actions reveal, use, resist, or change the world
- rules have limits and cannot solve every conflict
- each core character has agency beyond serving another character's arc
- relationships contain goals, leverage, dependency, or friction appropriate to the story
- no World Book and Character Card facts contradict each other
- every inferred item is visibly marked and reversible
- unresolved unknowns are either blocking or safely deferred

## Foundation Summary and Approval

Present:

```markdown
## Story Foundation Proposal

### Core Promise

### World Book

### Core Character Cards

### Relationship / Conflict Engine

### Verified

### Inferred — Approval Needed

### Unknown — Can Wait

### Blocking Decisions
```

Ask the user to approve, combine, or revise the proposal. Do not draft prose during Foundation setup.

After explicit approval:

1. mark the Foundation `approved`
2. create or propose updates to `story.md`, the World Book, and Character Cards
3. proceed to plot threads, chapter contracts, and outline work
4. retain the normal approval gate before original prose

## Genericity Boundary

Do not prefer any city, nation, era, genre, technology, magic system, species, relationship, identity, plot event, death, ending, or named inspiration. Use examples only when the user supplied them for the current project.

## Common Mistakes

- asking world questions when the user already provided a world
- forcing world-first setup onto a character-first seed
- generating a complete encyclopedia before a story engine exists
- treating proposed material as approved canon
- asking every template field as a separate question
- inventing dramatic secrets or deaths to make a card feel interesting
- repeating Foundation intake for an existing project
- drafting a sample chapter before Foundation approval

## Principle

Discover the story's foundation from the user's strongest seed, then make world and character constrain each other before plot begins.
