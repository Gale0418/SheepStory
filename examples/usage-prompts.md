# Usage Prompts

These prompts are designed to activate SheepStory reliably.

## 1. Start a New Story Project

```text
Use my-novel-writing to set up a new story project.
First ask for missing premise details.
Then propose a story bible, character files, worldbuilding files, plot files, and continuity files.
Do not draft prose.
```

## 2. Plan a Chapter

```text
Use my-novel-writing to plan the next chapter.
First reconstruct current continuity state.
Then produce a beat outline.
Each beat must include desire, obstacle, cost, turn, and irreversible change.
Do not write prose until I explicitly approve the outline.
```

## 3. Draft After Approval

```text
大綱 OK，照這個寫。
Use my-novel-writing to draft the chapter.
After drafting, run continuity, conflict pressure, dialogue, style preservation, and anti-AI-flavour checks before showing the final text.
```

## 4. Continuity Check

```text
Use my-novel-writing to continuity-check this outline.
Look for impossible timeline, character knowledge errors, emotional reset, object-state mistakes, unresolved promises, payoff before setup, and relationship changes that were not earned.
```

## 5. Make a Scene Less Peaceful

```text
Use my-novel-writing to revise this scene outline.
The scene currently feels too peaceful.
Add pressure through incompatible goals, hidden information, cost, pride, social consequence, or misread motives.
Do not add random violence unless it is already part of the story logic.
```

## 6. Dialogue Polish

```text
Use my-novel-writing to polish this dialogue.
Do not make the characters more polite.
For each important exchange, preserve intention, obstacle, subtext, and character voice.
Remove exposition disguised as dialogue.
```

## 7. Anti-AI-Flavour Polish

```text
Use my-novel-writing to remove AI flavour from this prose.
Preserve roughness, ambiguity, narrator rhythm, and character voice.
Do not over-polish.
```

## 8. Genius-vs-Genius Strategy

```text
Use my-novel-writing to design a genius-vs-genius confrontation.
Both sides must be competent.
List each side's known information, hidden information, resources, constraints, unacceptable cost, and blind spot.
Red-team the protagonist's plan before choosing the best available plan.
No perfect plan. No enemy stupidity.
```

## 9. Battlefield Anomaly

```text
Use my-novel-writing to plan a tactical scene where the protagonist's model fails.
Define expected enemy behavior, hidden anomaly, model failure, damage control, and after-action update.
The genius may fail, but the failure must teach them something.
```

## 10. Twist Budget

```text
Use my-novel-writing to review this chapter for twist budget.
Check whether every major reversal had setup, cost, permanent state change, and consequence scene.
Flag any washing-machine pacing.
```
