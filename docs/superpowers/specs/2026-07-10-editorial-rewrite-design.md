# SheepStory Editorial Rewrite Design

## Goal

Add reliable natural-language editing guidance without relying on vague instructions such as "humanize this" or turning SheepStory into a generic non-fiction editor by default.

## Alternatives

1. Extend `anti-ai-flavour.md`: compact, but mixes symptom detection with general-purpose rewrite behavior.
2. Add `references/editorial-rewrite.md`: keeps detection, voice evidence, preservation, and transformation separate. This is the selected approach.
3. Create another skill: improves independent discovery, but broadens the plugin and duplicates SheepStory's existing style infrastructure.

## Responsibilities

- `voice-calibration.md`: determine voice from explicit instructions and samples.
- `style-preservation.md`: identify what must survive the edit.
- `anti-ai-flavour.md`: detect plastic, templated, or mechanically balanced prose.
- `editorial-rewrite.md`: choose and execute an observable rewrite mode.

## Rewrite Contract

Before editing, identify the available audience, purpose, tone, locale, preservation constraints, and requested output. Use supplied information; do not interrogate the user when a bounded edit can be completed safely.

All modes must preserve claims, facts, stance, uncertainty, necessary qualifications, established terminology, and authorial markers. They must not add arguments, metaphors, examples, emotional claims, or stronger certainty unless requested.

If the source is already effective, make only material edits. Default to returning only the revised text when the user requests a direct rewrite; provide diagnosis or change notes only when requested or needed to explain a blocking ambiguity.

## Modes

- Natural: remove stiffness, repetition, template language, and predictable cadence.
- Concise: remove redundancy and inflated phrasing without deleting meaning or necessary qualifications.
- Plain language: reduce background knowledge and vocabulary burden without infantilizing adult readers.
- Conversational: make the text speakable and direct without forced slang, rhetorical questions, or fake intimacy.
- Voice-preserving: make the smallest changes needed while protecting rhythm, humor, roughness, dialect, and deliberate awkwardness.

Mode selection is based on explicit requests. When the user only says "humanize," default to voice-preserving natural editing and treat "humanize" as an imprecise goal label, not an editing operation.

## Scope

SheepStory loads this reference automatically for fiction, synopsis, pitch, lore, author notes, and narrative-adjacent editing. Generic non-fiction should not trigger SheepStory automatically, but an explicit request to use SheepStory may use the same reference.

## Testing

- Add a regression gate requiring the reference, routing, preservation contract, and five modes.
- Add positive-control specifications for claim preservation, adult plain language, and minimal no-op editing.
- Record the existing baseline behavior that improved flow but introduced unsupported implications.
- Re-run the same fresh-agent scenario after implementation and compare semantic preservation.

## Non-Goals

- Detecting whether text was generated by AI.
- Evading AI-content detectors.
- Guaranteeing that prose will be perceived as human.
- Adding a sixth overlapping "humanizer" mode.
